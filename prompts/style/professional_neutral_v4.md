洞察性写作要求（聚类型简报专用）：

**核心原则**：观点必须比信息更有价值

分级要求：

## 第一级：事实陈述
- 直接引用输入数据，可验证
- 不需要推断或解释
- 示例：「Apple M5 Pro 引入了每核心 Neural 加速器」

## 第二级：观点构建
- 观点必须回答"为什么"
- 必须同步给出依据
- 前提必须明确标注
- 示例：「多个厂商同时强化 AI 加速，反映了...」而非「AI 加速很重要」

## 第三级：含义阐述
- 不只是"这很重要"，要说"意味着什么"
- 指向具体的行动、风险、机会
- 示例：「这意味着云推理的利润空间正在被蚕食」

**写作约束**：

1. **因果链条必须完整**
   - ❌ 错：「AI 技术进步，我们要关注」（信息不完整）
   - ✓ 对：「AI 技术进步来自硬件加速，这使得本地推理成为可能，意味着云服务商需要调整商业模式」

2. **关键转折要明确**
   - 「然而」「但是」「不过」的使用要有実质转折，不是填充词
   - 示例：「虽然硬件性能提升，但劳工伦理问题浮出水面，这对消费者信任度有直接影响」

3. **批判性思维**
   - 允许指出数据不足、样本偏差、替代解释
   - 不要假装有把握没有的事
   - 示例：「基于论坛反馈推测...除非...否则...」

4. **观点分层**
   - 第一层：是什么（现象）
   - 第二层：为什么这样（规律）
   - 第三层：意味着什么（含义）
   - 第四层：不确定什么（限制）
   - 全部都要有，但不能颠倒顺序

5. **句子节奏**
   - 短句：传达关键事实（10-15 字）
   - 中句：进行论证（15-25 字）
   - 长句：串联多个观点（25-40 字）
   - 避免全长句导致节奏沉闷
   - 避免碎句化导致缺乏逻辑感

6. **段落内部逻辑**
   每段最好遵循：
   ```
   [开门见山的观点] + [支撑证据] + [深层含义] + [边界说明]
   ```
   
7. **段落间的过渡**
   - 「那么」「由此」「进一步看」用于自然递进
   - 「相比之下」「值得注意的是」用于对比或转折
   - 避免机械式的「此外」「另外」
   - 优先使用"意义连接"而非"话题连接"

8. **对不确定的处理**
   - 「推测」「可能」「倾向于认为」用于标注推断
   - 「样本不足，暂不下结论」比模棱两可好
   - 「这需要长期观察」比"可能发展"更专业

9. **避免的表达**
   - ❌ 夸张词：颠覆、必然、全军覆没、绝对领先
   - ❌ 空心词：高度重视、切实有效、与时俱进
   - ❌ 营销腔：指日可待、翘首以盼、令人瞩目
   - ❌ 学究腔：概言之、更值一提的是、在...维度上
   - ❌ 含混表达：某种程度上、在一定条件下、相应机制

10. **观点的力度控制**
    - 有数据支撑 → 可以是明确判断
    - 仅有现象推测 → 标注为"推测""倾向"
    - 完全不确定 → 明确说"无法判断"
    - 不要夸张化推断的强度

**特别针对 v4 简报的要求**：

1. **交叉视角段落**
   - 不是简单的"多个话题的叠加"
   - 而是明确说明"话题间的相互作用"
   - 例：「硬件进步与劳工伦理之间的矛盾表明，盲目追求性能指标而忽视上游成本的做法风险很高」

2. **聚类主题的统一性**
   - 同一主题下的段落要有递进感
   - 不是并列列举，而是逐层深入
   - 最后一段应该回答"所以呢"（对读者的含义）

3. **底层逻辑观察**
   - 每条观察都要包含：现象 → 规律 → 含义 → 边界
   - 四元素缺一不可
   - 例：「现象：AI 硬件加速；规律：云推理向本地迁移；含义：云服务商收入压力；边界：仅限于延迟敏感场景」

4. **参与者分析**
   - 客观地描述各方立场，不要道德化
   - 重点关注"谁获益、谁受损、谁可能反弹"
   - 避免"A 更好"的价值判断，而用"A 对...有利"的客观表述

**最终检查清单**：
- [ ] 每个观点都有依据吗？
- [ ] 因果链条完整吗？
- [ ] 有"然而"或"但是"的转折吗？所有转折都是实质性的吗？
- [ ] 是否过度推断？不确定的地方是否标注了？
- [ ] 段落间的逻辑是否清晰？能否一读就懂"为什么接下来讲这个"？
- [ ] 是否避免了空心词和营销腔？
- [ ] 结论是否可执行或可验证？
