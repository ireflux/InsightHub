## 新闻速递

### 镜中恶鬼，隐匿于代码深渊：I found 10k GitHub repositories distributing Trojan malware

一名安全研究员发现了一个大规模恶意软件分发活动。活动涉及 10,000 个 GitHub 仓库，它们由不同贡献者创建，并非其他仓库的 Fork。这些仓库的提交历史显示，它们会定期删除上一个提交并重新推送，唯一的改动是在 README 文件中添加指向压缩包的链接。压缩包内包含 Application.cmd、loader.exe 等文件，样本在 VirusTotal 上扫描显示 0 病毒，但解压后能检测出木马。尽管作者多次向 GitHub 支持提交删除请求，响应时间较长，但最终这些仓库已被移除。

### 瓦尔哈拉的晨曦：Java 28 的价值类型：Project Valhalla, Explained: How a Decade of Work Arrives in JDK 28

Oracle 工程师确认 Project Valhalla 的 JEP 401（值类与对象）已集成至 OpenJDK 主仓库，目标版本为 JDK 28。该变更包含超过 19.7 万行代码，涉及 1,816 个文件，导致社区要求暂时停止较大的提交。Brian Goetz 提醒，这是“第一部分”，目前为预览且默认禁用。Valhalla 的核心目标是实现“代码像类，运行像值”，即允许开发者编写具有类结构的代码，同时让 JVM 将其作为密集内存布局（类似结构体）处理，以提升缓存局部性并减少指针间接寻址的开销。

### 零延迟的推演：DuckDB 的内存魔术：DuckDB Internals: Why Is DuckDB Fast? (Part 1)

DuckDB 作为进程内分析型数据库，通过消除网络协议开销实现了高性能。与传统的服务型数据库不同，DuckDB 直接将库加载到进程中，查询结果在内存中直接处理，无需序列化传输。该技术路径引用了 2017 年的研究，指出客户端协议（如 ODBC/JDBC）往往是查询响应中最慢的环节。DuckDB 能够快速扫描 Parquet、CSV 等文件，这得益于其基于列式的存储设计和针对单节点大规模聚合优化的执行引擎。

### 芯片的裸露：MIT 研究员构建的 Fractal OS：To study how chips work, MIT researchers built their own operating system

为了研究芯片的真实行为，MIT 研究员开发了名为 Fractal 的环境。该系统被描述为一种操作系统，但评论指出其本质更接近于一个暴露芯片特性的引导程序。作者 Joseph 表示，这是内部开发的长期项目，旨在为 Spectre/Meltdown 等安全研究提供一个去除操作系统噪声的干净环境。相关代码与实验数据已在 GitHub 开源。

### 时间的颗粒：ClickHouse 十年开源之路：Ten years of ClickHouse in open source

ClickHouse 于 2016 年 6 月 15 日开源，至今已十年，拥有超过 2000 名贡献者。作者将开源分为四个等级，ClickHouse 已达到第三等级，具备透明的开发流程、任务追踪、代码审查及 CI 系统。文章详细回顾了从 2009 年的缓存优化实验到 2016 年正式开源的历程，强调了其作为 C++ 学习标杆和内存管理实验平台的地位。作者还介绍了如何通过系统表 `system.contributors` 感谢每一位贡献者。

### 悖论的修习：机器学习研究的 Zen：Zen and the Art of Machine Learning Research

AI 研究需要阅读与构建并重。文章认为，成功的灵感往往不可预测，坚持投入时间比天赋更重要。对于研究者，不应追逐过热的话题（如 2026 年的 Agents），而应回归基础数学（如 SVD、交叉熵）。文章指出，经验有时会成为阻碍，现代 AI 领域（如 OpenAI）的年轻团队证明了“初学者的心态”在快速变化的技术中反而是一种优势。建议避免只追求基准分数，而是寻找能真正测试新方法的数据集。

## 编辑手记

当前科技前沿呈现出一种“深度”与“广度”并存的态势。从 JDK 28 的 Valhalla 项目看，编程语言的设计必须紧跟硬件架构的演进，尤其是内存密度对现代 CPU 性能的决定性影响，这不仅是 Java 的挑战，也是所有语言设计者需要面对的物理极限。与此同时，DuckDB 和 ClickHouse 的崛起则证明了在分布式计算之外，单节点的高性能引擎依然具有巨大的实用价值，它们通过避开网络协议的瓶颈，在数据密集型应用中建立了新的效率标杆。

然而，繁荣的开源生态也伴随着阴暗面。GitHub 上万级恶意仓库的存在，暴露了托管平台在应对自动化恶意活动时的治理困境。虽然自动化脚本可以检测模式，但人工审核与响应的滞后性，使得恶意内容能够在较长时间内通过伪装存在，这对平台的安全机制提出了更高的要求。

在软件工程与 AI 研究领域，趋势正在回归本质。MIT 的 Fractal 项目和 ClickHouse 的开源实践都表明，理解底层机制（无论是芯片行为还是 C++ 内存管理）对于解决复杂问题至关重要。而 AI 研究的“Zen”则提醒从业者，在追逐 LLM 和智能体等热门概念时，不应忽视数学基础与原始数据的驱动力。
