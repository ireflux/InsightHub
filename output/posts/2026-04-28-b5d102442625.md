## 新闻速递

### 云端裂隙：微软终止与 OpenAI 的独家营收分成协议

微软和 OpenAI 结束了双方长达数年的独家营收分成协议。根据报道，微软将停止与 OpenAI 分享相关收入，OpenAI 也不再受限于仅在 Microsoft Azure 上运行。社区讨论中，有人认为这是 Google 可能成为最大赢家，因为 OpenAI 可能会转用 Google 的 TPU。也有观点指出，微软此前对 OpenAI 的限制实际上是在损害其投资价值，尤其是在 Anthropic 等竞争对手出现之后。目前尚不清楚 OpenAI 的股权比例是否也发生了相应变化。

### 光之囚笼：ASML 成为全球半导体制造的终极瓶颈

ASML 是唯一能制造极紫外光刻机（EUV）的公司，这些机器控制着尖端芯片的制造。这些机器极其复杂，包含超过十万组件，依赖与政府及竞争对手的合作。文章回顾了 ASML 如何通过押注当时尚不成熟的 EUV 技术，从芯片制造领域的落后者转变为垄断者。文章指出，这种垄断不仅体现在技术上，更成为了美中科技博弈的焦点。

### 时空回响：1930 年代语言模型 Talkie 模拟历史对话

Talkie 是一个基于 1930 年代文本训练的 13B 参数语言模型。该项目通过 24/7 直播展示该模型在缺乏现代知识背景下的行为。研究团队利用该模型测试预测未来事件的能力以及在没有数字计算机知识的情况下学习现代编程的能力。社区讨论展示了模型在回答关于 2025 年未来或 1926 年印度状况等问题时的表现，同时也指出了模型对某些历史事件（如大萧条）认知的局限性。

### 暗门：Linux 命令利用指南 GTFOBins 引发热议

GTFOBins 是一个列出可用于权限提升或绕过受限 shell 的 Linux 命令的网站。社区讨论集中在如何利用这些命令读取文件或执行代码，特别是在用户拥有受限 Shell 访问权限的情况下。有用户分享了他们在 CTF 或受限环境中使用类似技术的经验，认为这对安全研究者和红队很有价值。也有用户建议，除了展示如何绕过限制，文档还应包含相关的缓解措施。

### 伪栈机：WebAssembly 的栈机器假说被技术博客推翻

一篇技术博客声称 WebAssembly（Wasm）并非真正的栈机器，尽管 Wasm 规范和 Wikipedia 都是这样描述的。作者通过手写 Wasm 代码发现，Wasm 缺乏某些栈操作指令（如 `dup` 和 `rot`），这使得它与其他栈机器不同。文章解释了寄存器机器与栈机器在处理值重用时的区别，认为 Wasm 更接近一种混合机制。这一观点在开发者社区引发了对底层技术定义的重新审视。

### Rust 链接：pgrx 框架简化 Rust 编写 PostgreSQL 扩展

pgrx 是一个用于用 Rust 开发 PostgreSQL 扩展的框架，旨在提供类似 Rust 的惯用和安全性。该框架支持从 PostgreSQL 13 到 18 的多个版本，并提供了自动架构生成、内存管理和函数声明等特性。社区成员表示该项目推动了公司的发展，并讨论了在托管服务中使用自定义扩展的局限性。目前尚不清楚该框架是否会被主流云服务商广泛支持。

### 多余的烦恼：内存过量反致性能骤降

一篇怀旧技术文章讨论了早期 PC 硬件中的内存管理问题。作者在构建一台 1997 年的 Quake PC 时发现，安装超过特定量级的内存（如 96MB）会导致性能大幅下降。文章解释了 430FX 芯片组只能缓存 64MB 内存的限制，以及 Windows 加载机制如何导致软件运行在非缓存区域。社区讨论中，有人分享了使用内核补丁解决该问题的经验，也有人感叹现代开发者在处理资源管理上的便利。

## 编辑手记

微软与 OpenAI 关系的解绑是一个明确的战略信号，表明 OpenAI 正在寻求更多的自主权与资源灵活性。这一变动不仅可能重塑云服务市场的竞争格局，也可能促使 OpenAI 重新评估其计算基础设施的依赖路径。关于 Google TPU 在其中的角色，目前缺乏 OpenAI 官方确认的具体计划，但这确实是一个值得关注的潜在变数。

ASML 的垄断地位再次提醒我们，尖端制造技术的集中化是一把双刃剑。虽然它带来了极高的技术壁垒，但也使得全球半导体供应链极度脆弱。文章中提到的“替代社会设计”的思考虽然略带哲学意味，但也指出了技术依赖背后的系统性风险。如果地缘政治摩擦进一步激化，缺乏备用供应商的现状可能会成为全球经济的最大痛点。

技术领域的讨论往往在细节中涌现真知。Wasm 的“栈机器”定义修正和 GTFOBins 的安全利用讨论，都展示了开发者社区在深入理解技术细节时的活跃度。这些看似边缘的话题，实则反映了技术栈的复杂性与演进。对于读者而言，关注这些细节有助于在工程实践中避免潜在的陷阱，并在技术选型时做出更准确的判断。

接下来值得观察的具体信号是：OpenAI 在失去微软独家合作后的基础设施选择，特别是其是否会大规模采用 Google 的 TPU 或转向 AWS；以及 ASML 在地缘政治压力下，其技术出口管制对全球芯片供应链的直接影响。
