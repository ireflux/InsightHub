### [换脑记：DeepClaude – Claude Code agent loop with DeepSeek V4 Pro](https://github.com/aattaran/deepclaude)

Claude Code 的终端体验保持不变，但通过环境变量配置，开发者可以将 API 调用的后端从 Anthropic 切换至 DeepSeek V4 Pro。这种“换脑”操作允许用户继续使用 Claude Code 的文件编辑、Bash 执行和 Git 等工具链，仅改变底层的推理模型。

DeepSeek V4 Pro 在 LiveCodeBench 上得分 96.4%，输出 token 价格仅为 0.87 美元/百万，相比 Anthropic 的 15 美元/百万，成本降低了约 17 倍。工具提供了 `--backend` 参数，支持 DeepSeek、OpenRouter、Fireworks AI 以及原版 Claude Opus，并利用自动上下文缓存技术，在多轮对话中进一步降低开销。

有用户指出，DeepSeek 的低价是“75% 的补贴”，且补贴政策截止日期为 2026 年 5 月；同时，尽管 OpenRouter 声称服务器位于美国，但实际路由仍指向中国境内的 DeepSeek API。此外，由于 DeepSeek 的 Anthropic 兼容端不支持图像输入，该工具目前无法处理视觉任务。

### [伪官方：Trademark violation: Fake Notepad++ for Mac](https://notepad-plus-plus.org/news/npp-trademark-infringement/)

Notepad++ 项目创始人 Don Ho 警告称，名为 `notepad-plus-plus-mac.org` 的网站是虚假的，它未经授权使用 Notepad++ 商标，甚至伪造了创始人的个人资料以增加可信度。该网站声称提供官方 macOS 版本，但实际上 Notepad++ 历史上从未发布过 Mac 版本，任何声称该版本存在的人都是利用该名称进行诈骗。

Don Ho 表示，他已经联系了该网站所有者，但尚未收到回应。社区呼吁用户在 Reddit、Twitter 等平台看到类似宣传时，回复澄清这是未经授权的项目，并提供官方公告链接。

### [蓝图绘画：Using “underdrawings” for accurate text and numbers](https://samcollins.blog/underdrawings/)

为了解决多模态 AI 模型在生成图像时难以准确呈现文本和数字的问题，开发者提出了一种“underdrawing”技术。该方法分为两步：首先使用代码（如 SVG）生成一个结构确定、文字排版正确的底图，然后将该底图作为输入，结合提示词输入到多模态模型中生成最终图像。

这种方法通过先确定“骨架”（结构），再填充“血肉”（风格），显著提高了图像中文字的可读性和准确性。测试显示，该技术在生成螺旋排列的编号棋盘时，能完美保持数字的顺序和形状。不过，用户也指出，虽然文本准确，但模型在处理艺术风格细节时可能仍存在偏差，且该过程需要额外的编程步骤。
