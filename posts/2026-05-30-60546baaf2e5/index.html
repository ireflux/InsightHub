新闻速递

欧洲的 AI 堡垒：Notes from the Mistral AI Now Summit

Mistral AI 正在从单一的模型提供商转变为拥有完整 AI 生态的公司，涵盖计算、模型、平台和咨询。

该公司拥有位于巴黎的 40MW 数据中心，并计划在瑞典建立更多数据中心，强调开放、定制化和私有化部署。Mistral 宣称其小模型（如用于 OCR 的 Document AI 和多语言语音的 Voxtral）在效率和速度上优于大型通用模型。尽管获得银行（如 BNP Paribas 和 Abanca）的订单，部分社区成员认为 Mistral 在推理模型方面落后于 Gemma 和 Qwen 等竞争对手。

嵌入式巨兽：SQLite is all you need for durable workflows

SQLite 因其轻量级、低运维成本和高可靠性，正被越来越多地用于替代传统数据库来构建工作流。

工具 Temporal 等也采用 SQLite 作为本地安装的后端，简化了 API 重试和工作流的任务组织。讨论中存在分歧，支持者指出 SQLite 在 NVMe 磁盘和备份策略下，单线程写入仍可达到每秒数千次的性能，足以应对初创公司的大部分负载。批评者认为 SQLite 是嵌入式数据库，不适合多进程并发管理，建议关键生产应用应使用 Postgres 或 MySQL 等服务器级数据库。

估值悖论：Danish pension fund excludes SpaceX citing governance and valuation

丹麦养老基金 AkademikerPension 宣布从其投资组合中剔除 SpaceX，理由是治理结构和估值过高。

该基金此前已剔除了特斯拉以及从事武器和化石燃料生产的公司。分析显示，该基金在过去十年（2009-2021 年）的平均年回报率在丹麦同行中表现最佳。投资者对 SpaceX 的估值（约 1.8 万亿美元）提出质疑，讨论集中在其是否真的具备如此高的商业价值，还是仅依赖 Starlink 和火箭发射业务。

编译器的重构：Zig: Build System Reworked

Zig 项目对构建系统进行了重大重构，将负责构建图逻辑的“配置器”与负责执行构建的“制造者”分离为两个独立进程。

修改后，用户修改 build.zig 逻辑时，只有配置器进程会被重新编译，制造者进程可利用全局缓存保持优化状态。性能测试显示，在运行 zig build -h 命令时，重构后的耗时从约 150 毫秒显著下降至 14.3 毫秒，降幅超过 90%。这一改动旨在提升构建速度，减少不必要的重复计算，并使第三方工具更容易读取构建配置。

N64 的代码：Snowboard Kids 2 is 100% Decompiled

N64 游戏Snowboard Kids 2 实现了 100% 反编译，所有游戏函数均已用 C 语言重写并匹配原始汇编代码。

该项目历时约两年，社区贡献巨大，其中 AI 编码模型（如 Codex 和 GLM）在处理最难函数时发挥了重要作用。完整的反编译代码库使得游戏机制更加透明，为未来的重新编译、资产提取和模组制作奠定了基础。作者计划在修复 Bug 后发布高质量的重编译版本，并考虑将 Snowboard Kids 1 的代码与 2 的引擎合并。

编辑手记

从 Mistral 的转型和 SQLite 的流行可以看出，技术选型正从“大而全”向“小而美”和“本地可控”回归。Mistral 强调的“主权”和“私有化”正好切中了欧洲企业在面对美国科技巨头时的顾虑，BNP Paribas 和 Abanca 的合作案例证明了这种模式在受监管行业的可行性。与此同时，SQLite 在高并发场景下的争议反映了工程实践中“简单”与“复杂”的永恒博弈，尽管批评者指出了其并发限制，但其在运维成本和简单性上的优势依然吸引了大量开发者。

投资界对 SpaceX 的剔除也暗示了市场对高估值科技公司治理风险的重新评估。当一家公司的估值完全依赖于未来的技术突破和垄断潜力，而缺乏稳定的现金流支撑时，投资者可能会因治理问题而重新审视其价值。
